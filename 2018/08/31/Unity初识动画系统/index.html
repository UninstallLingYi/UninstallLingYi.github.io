Unity初识动画系统

发表于 2018-08-31

动画系统与TimeLine
打开动画窗口：window-Animation 来创建普通动画
Window-Animator来打开状态机
Animation窗
samples 60 可在此处设置x帧/秒 ，值越小越慢，值越大越快
Add Property 去添加关键帧物体的变化属性（关联组件）
在下方右键以Add Key 添加关键帧
右上角显示当前所在帧
使用录制功能可以免去许多冗杂的操作，直接在理想的时间点修改面板属性，将会直接进行保存//在使用录制功能时，无论如何都要在第0帧进行一次修改操作以保存初始状态
在曲线模式中，可以shift并滑轮以仅放大/小纵向
在曲线模式中，在关键帧右键以Broken破坏调节直线，Left/Right Tangent 调整 左/右 增减趋势
Free以自由调节，Linear以修改为直线
单击左上角的动画名，Add—以增加新的动画
动画循环：LoopTime勾选
UGUI的按钮状态
即UI-Button-Transition-Animation改为动画状态，Normal Highlighter（选中） Pressed（按下） Disabled（不可选），一般仅需要一帧动画，Dis一般为浅色，通常按键按下小默认大
2D精灵动画
得到资源后，若任意几个帧为一个动画，可直接拖入Hierarchy面板
人物角色的
Unity对于.fbx的支持是最佳的，@命名的一般为动画。在导入3D模型资源时，有大几率发生贴图或材质丢失，如何找回丢失的资源?
将角色拖入场景，搜索t：skinned mesh renderer/meshrenderer来查找renderer，Standard为标准，Albedo贴主图颜色为白色，Smothness默认为0（光滑度），Metallic金属色泽 Normal Map贴法图
在模型中的Scale Factor 中设置模型的倍数大小，点击Apply应用
Rig动画导入方式
Animation Type：Legacy少用且较为老旧，不支持状态机。Humanoid仅适用于人形模型 Avatar Definition骨骼相关（从本模型创建或从其他模型copy）//一般在多个模型共有同一骨架时copy，否则默认
Root node选择根节点 Optimize Game Objects优化游戏物体//这会导致模型的骨骼节点被隐藏，模型会更加简洁
Animations
Import Animation加载动画 Clips中Start和End可看出动画的帧数 修改名字后，在1.00处调整速度，可以拖拽黑标来修改，单击+来添加新的动画
Loop Time来循环，Loop pose（当匹配较为完美时，这会使动作较为优秀） 完成后单击Apply
在状态机中直接拖拽加入动画，第一个被拖入的为默认（可以在右键修改），右键“Make Transition”增加转换条件。HasExitTime为前一个动画结束后自动播放下一个，中间的间隔会使转换更为平滑（差值运算） 单击转换线并单击“-”来删除转换条件
在Parameters中添加对条件的限制参数，在转换线中Conditions中添加条件限制 由于参数是存在于状态机的，所以当访问修改参数的时候需要使用GetComponent（）;来获取组件，并建立相应类型的变量来接受这个组件。接受用户的输入（垂直轴：Vertical），调用变量中的Set方法[SetInteger为int]（要设置的参数,值的大小）;
动画的重定向：一个动画给多个模型使用。在Generic中无法实现，但是在Humanoid中可以实现
在Humanoid中可通过动画来控制移动，这会使动画和移动的协调性得到大幅提升
Configure只有在Apply后才能选定，用于配置Avatar，这会进入模型的骨骼场景，用于调试、更改Avatar映射。
Loop match来检测循环动画首尾一致性 Back into pose会使动画不再对该轴的数据产生影响 Based upon来计算原点 Body orientation为角色原点，即身体的正前方 Original是模型师设计的原点 Center of Mess质量的中心 Feet为两脚之间 Offset为偏移
在动画重定向时，若骨骼不同，则从自身创建，可以使用其他角色动画/状态机 ，只需要轻微调试。
